## Phase 1 — Jewellery fix (systemic + archetype)
- [x] **Root cause refined:** an existing `_MALE_EXCLUDED_VALUES` constraint system
      (`data/constraints.py`) already trims feminine-coded jewellery/nails/hair/makeup for
      `gender == "Male"`, but it ignored wardrobe (always trimmed). So the earlier `masculine_options`
      idea was redundant and couldn't honour "Feminine/Any wardrobe keeps options". **Reverted** the
      `fields.py`/`_build_option_pool` pool changes.
- [x] `data/constraints.py`: split `_MALE_EXCLUDED_VALUES` — jewellery/nails
      (earrings, necklace, other_jewelry, rings, bracelet, nails) rules flagged
      `presentation_gated=True`; hair/brows/lips/eye_shape/bust stay always-on (structural).
- [x] `nodes/identity_forge.py`: `_presentation_mode(gender, wardrobe)` helper (also refactors the
      outfit code); threaded `presentation` into `_apply_constraints`, which skips `presentation_gated`
      rules unless the man reads `"Masculine"`. Result: Match-gender male → masculine jewellery;
      Feminine/Any wardrobe male → full pool; females untouched. (Verified 0/400 vs 137/400 feminine.)
- [x] `data/templates.py`: locked `earrings`/`necklace` off on the anachronistic vintage-male
      archetypes. **Final list:** 1950s Suburban Dad, 1940s Swing Dancer, Classic Hollywood Leading
      Man, Roaring Twenties Gent, 1950s Greaser.
